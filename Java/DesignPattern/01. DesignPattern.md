## 01. DesignPattern

- 자주 사용하는 설계 패턴을 정형화시켜 가장 최적의 방법으로 정해 둔것
- 장점
  - 개발자간의 원활한 소통
  - 소프트웨어 구조 파악이 용이
  - 재사용을 통한 개발 시간 단축
- 단점
  - 초기 비용
  - 디자인패턴으로 구조/설계 필요

## 생성 패턴

- 객체를 생성하는 것과 관련된 패턴
- 객체의 생성과 변경이 전체 시스템에 미치는 영향을 최소화하고, 코드의 유연성을 높여준다.
- 종류
  - Factory Method
  - Singleton
  - Prototype
  - Builder
  - Abstract Factory
  - Chaining

## 구조 패턴

- 자료구조나 인터페이스 구조 등 프로그램 구조를 설계하는데 사용하는 패턴
- 클래스간의 구조를 개발하기 쉽도록 설계

- 종류
  - Adapter
  - Composite
  - Bridge
  - Decorator
  - Facade
  - Flyweight
  - Proxy

## 행위 패턴

- 반복적으로 사용되는 객체들의 상호작용을 패턴화 한 것
- 종류
  - Template Method
  - Interpreter
  - Iterator
  - Observer
  - Strategy
  - Visitor
  - Chain Of Responsibility
  - Command
  - Mediator
  - State
  - Memento
